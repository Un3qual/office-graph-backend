# API Migration Ledger

This ledger covers hand-written GraphQL and JSON API surfaces that remain live
while Office Graph migrates toward AshGraphql, AshJsonApi, and narrow custom
command or projection exceptions.

| Surface ID | Owner | Capability | Current surface | Exception class | Reason | Replacement target | Safety/parity tests | Retirement condition |
| --- | --- | --- | --- | --- | --- | --- | --- | --- |
| `graphql.query.health` | `OfficeGraphWeb` | GraphQL liveness | `OfficeGraphWeb.GraphQL.Common.Queries.health` | compatibility endpoint | Simple hand-written liveness field, not resource-backed. | Keep as compatibility or move to a dedicated health endpoint. | GraphQL smoke coverage when schema tests are updated. | Remove from product schema when generated GraphQL root is mounted or health moves outside product GraphQL. |
| `graphql.query.operator_inbox` | `OfficeGraph.Projections` | Operator workflow projection | `OfficeGraphWeb.GraphQL.OperatorWorkflow.Queries.operator_inbox` | projection exception | Mixed policy-filtered projection with actionability, watermarks, and product status. | GraphQL projection contract backed by `OfficeGraph.Projections.operator_inbox/1`. | `test/office_graph_web/operator_workflow_api_test.exs`. | Retire manual field only when generated reads or a modular GraphQL projection field replaces it with parity. |
| `graphql.query.operator_workflow_item` | `OfficeGraph.Projections` | Operator workflow projection | `OfficeGraphWeb.GraphQL.OperatorWorkflow.Queries.operator_workflow_item` | projection exception | Mixed projection over normalized intake, proposal status, graph links, audit, and revision traces. | GraphQL projection contract for operator item details. | `test/office_graph_web/operator_workflow_api_test.exs`. | Retire manual field when durable item projection module replaces monolithic schema field. |
| `graphql.query.operator_packet_readiness` | `OfficeGraph.Projections` | Packet readiness projection | `OfficeGraphWeb.GraphQL.OperatorWorkflow.Queries.operator_packet_readiness` | projection exception | Derived command-affordance projection over draft input, not a plain resource read. | Backend-owned readiness projection and command affordance contract. | `test/office_graph_web/operator_workflow_api_test.exs`. | Retire manual field when readiness projection moves into modular GraphQL or generated plus custom projection replacement. |
| `graphql.query.operator_run_state` | `OfficeGraph.Projections` | Run state projection | `OfficeGraphWeb.GraphQL.OperatorWorkflow.Queries.operator_run_state` | projection exception | Aggregate read across packet, run, checks, observations, evidence, and verification results. | Run/evidence GraphQL projection contract with generated resource reads where safe. | `test/office_graph_web/operator_workflow_api_test.exs`. | Retire manual field when the operator frontend consumes the replacement GraphQL projection. |
| `graphql.query.operator_verification_outcome` | `OfficeGraph.Projections` | Verification outcome projection | `OfficeGraphWeb.GraphQL.OperatorWorkflow.Queries.operator_verification_outcome` | projection exception | Derived verification outcome projection, not a single resource read. | Fold into run-state projection or replace with modular verification projection. | Add focused parity if this field remains beyond modularization. | Retire when run-state or verification projection exposes the desired outcome. |
| `graphql.mutation.execute_packet_run_verification` | `OfficeGraph.PacketRunVerification` | Packet-run-verification workflow | `OfficeGraphWeb.GraphQL.PacketRunVerification.Mutations.execute_packet_run_verification` | temporary compatibility path | One-shot flow remains a compatibility wrapper over the domain command that spans packet preparation, run start, observation, evidence suggestion, evidence acceptance, and verification summary. | Smaller command/projection surfaces or deletion once the product frontend no longer needs one-shot execution. | `test/office_graph_web/packet_run_verification_api_test.exs`, `test/office_graph/packet_run_verification_test.exs`. | Delete after desired callers use durable command/projection surfaces or the workflow no longer needs a one-shot entrypoint. |
