# Architecture Exception Ledger

This ledger records the remaining approved direct Ecto mutation paths after the
WorkGraph Ash repair. Normal walking-loop product resources should move through
Ash resources and domain actions. Entries here are explicit exceptions that keep
graph identity, relationship, bootstrap, intake, proposed-change, and
maintenance writes visible until a later change narrows or retires them.

| File | Owner | Approved functions | Allowed operation type | Approving spec | Reason | Retirement condition |
| --- | --- | --- | --- | --- | --- | --- |
| `lib/office_graph/work_graph.ex` | `OfficeGraph.WorkGraph` | `{graph_transaction/1, Repo.transaction}` | Transaction boundary only | `repair-ash-model-conformance/backend-model-ownership` | WorkGraph graph identity, relationship, typed resource, and document writes still share one explicit transaction boundary so partially-created graph state is not committed. Table-backed WorkGraph records now use Ash actions inside that boundary. | Retire or narrow this entry when WorkGraph creation is expressed as a fully Ash-managed atomic workflow. |
| `lib/office_graph/integrations.ex` | `OfficeGraph.Integrations` | `{insert_manual_intake/4, Repo.transaction}`, `{insert_source_then_refetch/1, Repo.insert_all}` | Transaction boundary plus idempotent conflict insert | `repair-ash-model-conformance/backend-model-ownership` | Manual intake creates or reads integration source, raw archive, and normalized replay/idempotency records through Ash actions inside one explicit transaction boundary before proposed graph changes are applied. External source creation uses one `ON CONFLICT DO NOTHING` insert plus an Ash refetch so first concurrent intakes sharing a new source do not poison the transaction on a unique-key race. | Retire or narrow this entry when manual intake storage is expressed as a fully Ash-managed atomic workflow or Ash upsert handles this external-source race without direct SQL. |
